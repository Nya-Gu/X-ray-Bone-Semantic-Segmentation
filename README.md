## 0. 프로젝트 소개 및 목차
**(1) 개요**: 손을 촬영한 X-ray 이미지에서 29개의 손가락, 손목, 팔뼈를 세그멘테이션하는 모델을 개발하는 프로젝트입니다.

**(2) 목표**: 모델 성능 평가 지표는 class mean dice로, 29개의 클래스의 dice를 평균 낸 값을 높이는 것이 이번 프로젝트의 목표입니다.

## 1. 베이스라인 실행 및 학습 관찰

우선 U-Net + EfficientNetB7 모델을 사용해 베이스라인을 돌려봤습니다. 모델의 학습 과정을 보면서 재밌는 점을 발견할 수 있었습니다. 모델은 학습 중 모든 클래스의 Dice를 골고루 높이는 방식으로 학습하지 않았습니다. 특정 클래스의 dice를 높이는 데 집중하다가 그것이 일정 수준 이상이 되면 그때부터 다른 클래스의 dice를 높이는 방식으로 학습이 진행되었습니다. 학습 초반에는 주로 Radius, Ulna와 같은 큰 뼈가 학습되었고 중반에는 긴 손가락 뼈와 손목 뼈가 학습되었으며, 후반에서는 손가락 끝 작은 뼈와 맞추기 어려운 손목뼈를 학습되었습니다. 아래 테이블을 참고하시면 좋을 것 같습니다.

<img width="2005" height="986" alt="image" src="https://github.com/user-attachments/assets/c705c58e-b8e7-4385-aedf-230371c84963" />

모델이 순차적으로 각 클래스를 학습하는 걸 보면서 팔뼈, 손목뼈, 손가락뼈 전문 세그멘테이션 모델을 학습시키는 게 효율적이라는 생각이 들었습니다. 이번 프로젝트에서 손가락뼈, 손목뼈 클래스의 Dice를 높이기 위해 수차례 모델 학습을 진행하게 될 텐데, 그때마다 모델이 이미 잘 맞추는 팔뼈 클래스를 다시 학습하게 하는 건 시간 소모적인 일일 것 같았습니다. 그리고 이미지의 사이즈가 2048x2048이어서 메모리가 부족한 상황이었는데, 팔뼈, 손목뼈, 손가락뼈 전문 모델을 학습시킨다면 이미지에서 팔뼈, 손목뼈, 손가락뼈 부위만 Crop해서 모델이 작은 사이즈의 이미지를 학습하면 되니 메모리 부담을 덜 수 있을 것이었습니다. 그래서 아래와 같이 이미지를 부위별로 Crop하고 전문가 모델을 학습시키기로 했습니다.

<img width="1909" height="716" alt="image" src="https://github.com/user-attachments/assets/165d2ab9-72ec-4696-82bd-dfea2a37b8ea" />

훈련 데이터는 위와 같이 정답 라벨과 X-ray 이미지가 한 쌍으로 있기 때문에 정답 라벨을 활용해 부위별 Crop 이미지를 쉽게 확보할 수 있지만, 현실의 X-ray 이미지에는 정답 라벨이 같이 있지 않아 부위별 Crop 이미지를 구하기 어려울 수 있습니다. 이 문제는 아래처럼 (1) 팔뼈, 손목뼈, 손가락뼈를 마스킹하는 모델(사실상 분류 클래스가 3개인 세그멘테이션 모델)을 학습시키고 (2) 그로부터 얻은 마스크로 각 부위의 중심 좌표를 계산해 Crop하는 방식으로 해결했습니다.

<img width="1841" height="832" alt="image" src="https://github.com/user-attachments/assets/5f1a27d1-2a1a-48e2-9a6b-59b784257845" />

처음에는 마스크의 최상단, 최하단, 최좌단, 최우단 좌표의 가운데 좌표로 중심점을 계산했습니다. 하지만 모델이 출력한 마스크에서 검은 배경이어야 할 곳에 하얀 마스크 픽셀이 찍히는 경우(FP 에러)가 있었고 이로 인해 Crop한 이미지에 각 부위가 제대로 담기지 않을 경우가 있었습니다. 그래서 중심 좌표을 계산할 때 FP 에러의 영향을 덜 받도록 하얀 마스크 픽셀들의 평균 좌표를 사용하기로 했습니다.

## 2. 데이터 관찰

다음으로 데이터를 관찰했습니다. 바로 눈에 보인 것은 뼈 주위 연부조직으로 인한 X-ray 산란 노이즈였습니다. (이를 Soft Tissue Effect라고 부르겠습니다.) 해당 노이즈로 인해 손가락 끝에 위치한 뼈와 손목 내부 뼈, 팔뼈의 윤곽을 제대로 파악하기 어려웠습니다. 이를 제거하면 모델 성능이 개선될 것이라 기대했고, 아래 논문의 방법을 사용해 노이즈를 제거하기로 했습니다.

Decompose X-ray Images for Bone and Soft Tissue([논문](https://arxiv.org/abs/2007.14510))([코드](https://pmc.ncbi.nlm.nih.gov/articles/PMC12433621/))

그 결과 아래에서 보이듯이 X-ray 이미지에서 Soft Tissue Effect를 어느 정도 제거할 수 있었습니다. (이렇게 Soft Tissue Effect를 제거한 이미지를 BTD 이미지라고 부르겠습니다.) 뒤에서 이야기할 내용이지만 BTD 이미지 사용이 항상 모델 성능을 개선하지는 않았습니다. Soft Tissue Effect를 제거하는 과정에서 데이터 타입 변환으로 인해 정보 손실이 발생하거나 미세한 아티팩트가 생성되는 이슈로 인해 오히려 모델 성능이 떨어지는 일이 발생하기도 했습니다. 결과적으로 저는 이 BTD 이미지를 팔뼈 전문 모델을 학습시킬 때에만 사용하기로 했습니다. 자세한 결과는 뒤에서 이야기하겠습니다.

## 4. 손가락뼈 모델 성능 개선

### (1) Dice box plot

손가락뼈 모델의 검증 데이터셋 추론 결과를 토대로 클래스별 Dice box plot을 그려봤습니다. 노란 점선은 각 뼈가 어느 손가락(엄지, 검지, 중지, 약지, 새끼)에 속한 뼈인지 쉽게 확인할 수 있게 하기 위해 그었습니다. (1,2,3)번 클래스는 엄지손가락 뼈, (4,5,6,7)번 클래스는 검지 뼈, (8,9,10,11)번 클래스는 중지 뼈라고 보시면 됩니다.

<img width="1161" height="638" alt="image" src="https://github.com/user-attachments/assets/63ad1d98-e9eb-447b-bb0a-d3c07ff4c741" />  

플롯에서 패턴이 보입니다. 각 손가락에서 상대적으로 작은 뼈(예: 1, 4, 5, 8번 클래스)의 dice는 상대적으로 큰 뼈(예: 2, 3, 6, 7 클래스)의 dice보다 낮은 경향을 보였습니다. 손가락에 끝에 위치할수록 뼈의 경계가 Soft Tissue Effect에 의해 흐려지기 때문에 모델이 제대로 세그멘테이션하는 데 어려움을 겪은 것으로 보입니다. 그래서 이를 해결하기 위해 Soft Tissue Effect를 제거한 BTD 이미지로 모델을 학습시켰습니다만, 안타깝게도 유의미한 성능 개선을 이루진 못했습니다.

### (2) FP/FN ratio box plot

다음으로 모델 성능 저하에 FP 에러와 FN 에러 중 어느 것이 더 기여했는지를 파악하기 위해 FP 에러와 FN 에러의 비율을 시각화했습니다.

<img width="1646" height="584" alt="image" src="https://github.com/user-attachments/assets/409e2fea-f8a0-4470-9d7a-a2eac1939c67" />  

그래프를 해석하는 방법은 다음과 같습니다.  
50th percentile를 나타내는 주황색 선이  
(1) 0(녹색 선)에 가까울수록, 해당 클래스의 Dice 성능 저하에 FP 에러와 FN 에러가 기여하는 정도가 비슷했음을 의미합니다.  
(2) 0보다 위에 위치할수록, 해당 클래스의 Dice 성능 저하에 FP 에러가 더 큰 영향을 주었음을 의미합니다.  
(3) 0보다 아래에 위치할수록, 해당 클래스의 Dice 성능 저하에 FN 에러가 더 큰 영향을 주었음을 의미합니다.   

손가락뼈 모델의 경우 FN 에러에 의한 성능 하락이 지배적인 모습을 볼 수 있었습니다. 이를 개선하기 위해서 Tversky loss를 사용하고 최적의 Threshold 값을 탐색했습니다.

<img width="1613" height="807" alt="image" src="https://github.com/user-attachments/assets/8842e104-ba9b-4eb6-8063-5bdd3564e0d0" />

Dice loss(=Tversky(α=0.5, β=0.5)) 대신 Tversky(α=0.45, β=0.55)를 사용했을 때, 다시 말해 FN 에러에 더 패널티를 부과하는 손실 함수를 사용했을 때, 모델 성능이 97.93에서 97.99%로 개선되는 것을 확인할 수 있었습니다. 또한 Treshold를 0.4로 설정함으로써 모델 성능을 98.00%로 개선할 수 있었습니다.

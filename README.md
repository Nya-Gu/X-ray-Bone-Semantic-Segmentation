# X-ray Bone Semantic Segmentation

## 0. 프로젝트 소개 및 목차
**(1) 개요**: 손을 촬영한 X-ray 이미지에서 29개의 손가락, 손목, 팔뼈를 세그멘테이션하는 모델을 개발하는 프로젝트입니다.

**(2) 목표**: 모델 성능 평가 지표는 class mean dice로, 29개의 클래스의 dice를 평균 낸 값을 높이는 것이 이번 프로젝트의 목표입니다.

**(5) 첨언**

dice는 0~1 사이의 값이지만 소수점을 그대로 표기할 경우 (그리고 소수점이 길어질수록) 읽기 불편한 면이 있는 것 같습니다. 그래서 편의상 100을 곱해 %처럼 표기하기로 했습니다.

예시: dice=0.9903 -> 99.03%

## 1. 베이스라인 실행 및 학습 관찰

우선 U-Net + EfficientNetB7 모델을 사용해 베이스라인을 돌려봤습니다. 모델의 학습 과정을 보면서 재밌는 점을 발견할 수 있었습니다. 모델은 학습 중 모든 클래스의 Dice를 골고루 높이는 방식으로 학습하지 않았습니다. 특정 클래스의 dice를 높이는 데 집중하다가 그것이 일정 수준 이상이 되면 그때부터 다른 클래스의 dice를 높이는 방식으로 학습이 진행되었습니다. 학습 초반에는 주로 Radius, Ulna와 같은 큰 뼈가 학습되었고 중반에는 긴 손가락 뼈와 손목 뼈가 학습되었으며, 후반에서는 손가락 끝 작은 뼈와 맞추기 어려운 손목뼈를 학습되었습니다. 아래 테이블을 참고하시면 좋을 것 같습니다.

<img width="2005" height="986" alt="image" src="https://github.com/user-attachments/assets/c705c58e-b8e7-4385-aedf-230371c84963" />

모델이 순차적으로 각 클래스를 학습하는 걸 보면서 팔뼈, 손목뼈, 손가락뼈 전문 세그멘테이션 모델을 학습시키는 게 효율적이라는 생각이 들었습니다. 이번 프로젝트에서 손가락뼈, 손목뼈 클래스의 Dice를 높이기 위해 수차례 모델 학습을 진행하게 될 텐데, 그때마다 모델이 이미 잘 맞추는 팔뼈 클래스를 다시 학습하게 하는 건 시간적으로 소모적인 일이 될 것이었습니다. 또한 이미지의 사이즈가 2048x2048로 커서 메모리 부족 문제가 발생하기 쉬운 상황이었는데 팔뼈, 손목뼈, 손가락뼈 전문 모델을 학습시킨다면 이미지에서 팔뼈, 손목뼈, 손가락뼈 부위만 Crop해서 모델이 작은 사이즈의 이미지를 학습하면 되니 메모리 부담을 덜 수 있을 것이었습니다. 그래서 저는 아래와 같이 이미지를 부위별로 Crop하고 전문가 모델을 학습시키기로 했습니다.

<img width="1909" height="716" alt="image" src="https://github.com/user-attachments/assets/165d2ab9-72ec-4696-82bd-dfea2a37b8ea" />

훈련 데이터는 위와 같이 정답 라벨과 X-ray 이미지가 한 쌍으로 있기 때문에 이를 활용해 부위별 중심점을 찾아 Crop을 수행할 수 있지만, 현실에서 만나는 X-ray 이미지에는 정답 라벨이 없어 각 부위를 Crop하기 어려울 것입니다. 이 문제는 아래처럼 (1) 팔뼈, 손목뼈, 손가락뼈를 마스킹하는 모델(사실상 분류 클래스가 1개인 세그멘테이션 모델)을 따로 학습시키고 (2) 그로부터 얻은 마스크로 각 부위의 중심 좌표를 계산해 Crop하는 방식으로 해결했습니다.

<img width="1841" height="832" alt="image" src="https://github.com/user-attachments/assets/5f1a27d1-2a1a-48e2-9a6b-59b784257845" />

처음에는 마스크의 최상단, 최하단, 최좌단, 최우단 좌표로부터 중심점을 계산했습니다. 하지만 모델이 출력한 마스크에서 검은 배경이어야 할 곳에 하얀 마스크 픽셀이 찍히는 경우(FP 에러)가 있었고 이로 인해 Crop한 이미지에 각 부위가 제대로 담기지 않을 경우가 있었습니다. 그래서 중심 좌표을 계산할 때 FP 에러의 영향을 덜 받도록 마스크 픽셀의 평균 x좌표와 y좌표를 구하기로 했습니다.

## 2. 데이터 관찰

다음으로 데이터를 관찰했습니다. 바로 눈에 들어온 것은 뼈 주위 연부조직으로 인한 X-ray 산란 노이즈였습니다. (이를 Soft Tissue Effect라고 부르겠습니다.) 해당 노이즈로 인해 손가락 끝에 위치한 뼈와 손목 내부 뼈, 팔뼈의 윤곽을 제대로 파악하기 어려웠습니다. 이를 제거하면 모델 성능이 개선될 거라고 기대했고 관련해서 방법을 모색하다가 아래 논문의 방법을 쓰기로 했습니다.

Decompose X-ray Images for Bone and Soft Tissue([논문](https://arxiv.org/abs/2007.14510))([코드](https://pmc.ncbi.nlm.nih.gov/articles/PMC12433621/))

그 결과 아래에서 보이듯이 X-ray 이미지에서 Soft Tissue Effect를 어느 정도 제거할 수 있었습니다. (이렇게 Soft Tissue Effect를 제거한 이미지를 BTD 이미지라고 부르겠습니다.) 뒤에서 이야기할 내용이지만 BTD 이미지 사용이 항상 모델 성능을 개선하지는 않았습니다. Soft Tissue Effect를 제거하는 과정에서 데이터 타입 변환으로 인해 정보 손실이 발생하거나 미세한 아티팩트가 생성되는 이슈로 인해 오히려 모델 성능이 떨어지는 일이 발생하기도 했습니다. 결과적으로 전 이 BTD 이미지를 팔뼈 전문 모델을 학습시킬 때만 사용하기로 했습니다. 뒤에서 이에 대해 더 이야기하겠습니다.

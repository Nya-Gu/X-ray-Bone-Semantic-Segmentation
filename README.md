## 1. 베이스라인 실행 및 학습 로그 관찰

- 우선 U-Net + EfficientNetB7 모델을 사용해 베이스라인을 돌렸습니다. 학습 로그 관찰 결과, 모델의 학습은 모든 클래스의 Dice 계수를 균등하게 향상시키는 방식으로 진행되지 않았습니다. 특정 클래스의 Dice를 우선적으로 개선한 뒤, 해당 성능이 일정 수준에 도달하면 이후 다른 클래스의 성능을 높이는 방향으로 학습이 이루어지는 경향을 보였습니다.

- 학습 초반에는 Radius, Ulna와 같은 비교적 큰 팔뼈를 중심으로 학습이 진행되었고 중반에는 긴 손가락뼈와 손목뼈로 확장되었습니다. 이후 학습 후반에는 손가락 끝의 작은 뼈, 구분이 어려운 손목뼈를 점진적으로 학습하는 양상을 보였습니다.

## 2. 프로젝트 수행 전략 수립

- Radius, Ulna와 같은 팔뼈 클래스는 Dice 계수가 약 0.98로, 모델이 이미 잘 맞추는 클래스였습니다. 따라서 성능 개선의 핵심은 상대적으로 예측이 어려운 손가락과 손목 부위에 있다고 판단했습니다. 이에 따라, 이미 잘 예측되는 팔뼈 클래스에 반복적으로 학습 자원을 소모하기보다 특정 부위에 집중한 모델을 별도로 구성하는 것이 프로젝트의 효율을 높이는 데 유리하다고 판단되었습니다.

- X-ray 이미지의 해상도는 2048×2048로 높은 편이어서 메모리 제약으로 인해 배치사이즈를 충분히 확보하기 어려웠습니다. 이를 해결하기 위해 이미지를 부위별로 Crop하여 학습하는 방식을 적용하면 이미지 해상도를 유지하면서도 메모리 사용량을 줄일 수 있어 효율적인 학습이 가능할 것이라고 판단했습니다. 

- 이러한 판단을 바탕으로 전체 태스크를 손가락, 손목, 팔뼈 세 가지 클래스 세그멘테이션으로 분할하고 아래와 같이 각 부위를 독립적으로 학습하는 모델을 설계하기로 했습니다. 

<img width="1634" height="679" alt="image" src="https://github.com/user-attachments/assets/e793b1c1-7bea-4c57-8829-a1f8abb4d636" />

- 훈련 데이터는 정답 라벨을 활용해 각 부위의 이미지를 쉽게 Crop할 수 있지만, 현실의 X-ray 이미지에는 정답 라벨이 붙어 있지 않아 각 부위를 Crop하기 어려운 문제가 있었습니다.

- 이 문제는 아래와 같이 (1) 팔뼈, 손목뼈, 손가락뼈를 마스킹하는 모델(사실상 분류 클래스가 3개인 세그멘테이션 모델)을 학습시키고, (2) 이로부터 얻은 마스크에서 각 부위의 중심 좌표를 계산해 이미지를 Crop하는 방식으로 해결했습니다.

<img width="1841" height="832" alt="image" src="https://github.com/user-attachments/assets/5f1a27d1-2a1a-48e2-9a6b-59b784257845" />

- 처음에는 모델이 생성한 마스크의 최상단, 최하단, 최좌단, 최우단 좌표의 가운데 지점을 중심 좌표로 계산했습니다.

- 하지만 검은 배경이어야 할 곳에 마스크가 생성되는 경우, 다시 말해 외딴 곳에 False Positive 에러가 발생하는 경우, 중심 좌표가 제대로 계산되지 않아 Crop한 이미지 내에 각 부위가 제대로 담기지 않는 문제가 있었습니다.

- 따라서 중심 좌표을 계산 시, FP 에러에 의한 영향을 최소화할 수 있도록, 다시 말해 노이즈에 강건한 중심 좌표를 얻을 수 있도록, 위처럼 마스크 픽셀의 평균 x, y좌표를 중심 좌표로 쓰기로 했습니다.

## 3. 태스크 분할의 효과

- 태스크를 분할한 이후 배치 사이즈를 기존 4에서 16~32까지 안정적으로 확장할 수 있었고, 이에 따라 Batch Normalization의 통계량을 보다 안정적으로 확보할 수 있었습니다.

- 태스크 분할 전 모델의 학습 시간과 분할 후 모델들의 학습 시간의 총합은 거의 동일했습니다. 그러나 특정 클래스에 대한 성능 튜닝 시, 기존에는 약 160분이 소요되던 실험이 35~80분 수준으로 단축되는 이득이 있었습니다. 이러한 구조적 이점 덕분에 이후 100회 이상의 튜닝 실험(손가락 25회, 손목 51회, 팔 28회)에서 상당한 시간 절감 효과를 거둘 수 있었습니다.

|:---------:|:-:|:----:|:-:|:-:|
|학습 시간|160분|80분 (50%⬇️)|45분 (72%⬇️)|35분 (79%⬇️)|

## 4. 데이터 관찰 및 노이즈 제거 전처리

- 데이터를 관찰하면서 뼈 주변 연부조직으로 인한 X-ray 산란 노이즈에 주목했습니다. X-ray 산란 노이즈로 인해 경계가 흐려지는 손가락 말단, 손목 내부, 팔뼈 인접 부위에서 FN(False Negative) 오류가 빈번히 발생하는 것을 보고 노이즈가 FN 에러 발생에 기여하고 있다는 가설을 세웠습니다.

- 이를 검증하기 위해 해당 노이즈를 제거했을 때 모델 성능이 어떻게 변화하는지 실험적으로 확인하고자 했습니다. 노이즈 제거를 위한 다양한 방법을 검토하였고, 최종적으로 아래 논문의 방법론을 사용해 노이즈를 제거하기로 했습니다.

## 6. 손가락뼈 모델 성능 개선

### (1) Dice box plot

- 손가락뼈 모델의 검증 데이터셋 추론 결과를 토대로 클래스별 Dice box plot을 그렸습니다. 노란 점선은 각 뼈가 어느 손가락(엄지, 검지, 중지, 약지, 새끼)에 속하는지 쉽게 구분할 수 있도록 그어놨습니다. (1,2,3)번 클래스는 엄지 뼈, (4,5,6,7)번 클래스는 검지 뼈, (8,9,10,11)번 클래스는 중지 뼈라고 보시면 됩니다.

<img width="1747" height="638" alt="image" src="https://github.com/user-attachments/assets/0eabef9e-e616-4fee-9a3f-02b7a7cf9c20" /> 

- 각 손가락에서 상대적으로 작은 뼈(예: 1, 4, 5, 8번 클래스)의 dice는 상대적으로 큰 뼈(예: 2, 3, 6, 7 클래스)의 dice보다 낮은 경향을 보였습니다. X-ray 산란 노이즈에 의해 손가락 끝에 위치한 뼈의 윤곽의 흐려져 모델 성능이 저하된 것으로 보였습니다만, 노이즈를 제거한 이미지을 사용해 모델 학습을 수행해도 유의미한 성능 개선을 관찰할 순 없었습니다.

### (2) FP/FN ratio box plot

- 다음으로 모델 성능 저하에 있어 FP 에러와 FN 에러 중 어느 요인이 더 크게 기여했는지 파악하기 위해 추론 결과를 기반으로 두 에러의 비율을 시각화했습니다.

<img width="1646" height="584" alt="image" src="https://github.com/user-attachments/assets/409e2fea-f8a0-4470-9d7a-a2eac1939c67" />    

- 그래프를 해석하는 방법은 다음과 같습니다. 50th percentile을 나타내는 주황색 선이,  
(1) 0(녹색 선)에 가까울수록 FP와 FN 에러가 성능 저하에 유사한 수준으로 기여했음을 의미합니다.  
(2) 0(녹색 선)보다 위에 위치할수록 FP 에러가 성능 저하에 더 크게 기여했음을 의미합니다.  
(3) 0(녹색 선)보다 아래에 위치할수록 FN 에러가 성능 저하에 더 크게 기여했음을 의미합니다.  

 - 해당 플롯은 손가락뼈 모델의 결과로, FN 오류에 의한 성능 저하가 지배적인 경향을 보였습니다. 이를 개선하기 위해 Tversky loss와 Threshold 값을 중심으로 최적화를 진행하기로 했습니다.

- Dice loss(=Tversky(α=0.5, β=0.5)) 대신 Tversky loss(α=0.45, β=0.55)를 적용하여 FN 에러에 더 큰 패널티를 부과한 결과, 손가락뼈 모델의 성능을 0.9793에서 0.9799로 개선할 수 있었습니다. 또한 threshold를 0.4로 조정함으로써 손가락뼈 모델의 성능을 최종적으로 0.9800까지 향상시킬 수 있었습니다.

<img width="1703" height="694" alt="image" src="https://github.com/user-attachments/assets/dd780077-2eb1-458a-870d-ab29b9f5f54d" />

## 10. 프로젝트 후기

- 이전에는 모델 성능이 기대에 미치지 못할 경우, 그 원인과 해결 방법을 생성형 AI와 같은 외부 도구에 의존해 탐색하는 경우가 많았습니다. 그러나 이번 프로젝트에서는 이러한 접근 방식에서 벗어나 문제를 스스로 정의하고 해결하는 데 집중했습니다. X-ray 산란 노이즈 제거 방법을 직접 탐색하고, 추론 결과를 기반으로 FP/FN 에러를 분석했으며, Dice 및 FP/FN ratio box plot을 통해 성능 저하의 원인을 정량적으로 파악했습니다. 또한 이를 바탕으로 Tversky loss의 하이퍼파라미터와 threshold를 최적화하여 모델 성능을 개선할 수 있었습니다.

- 이러한 일련의 과정을 통해 기술적으로 성장하고 문제 해결을 위한 사고력과 분석 역량이 한층 강화되었음을 느낄 수 있었습니다. 성장 측면에서 유의미한 프로젝트였습니다.
